feat(phase3): Async pathfinding manager and smooth movement system

Implements Phase 3 (Pathfinding & Entity Movement) with async pathfinding,
path caching, and smooth hex-to-hex movement animation with turn-based
integration and real-time path visualization.

## New Components

### PathfindingManager.cs (328 lines)
- Singleton coroutine-based async pathfinding manager
- Request queue system with unique IDs and priority support
- Path caching with 5-second expiration and 100-path limit
- Performance throttling: 100 nodes/frame, 5ms/frame limit
- Cache invalidation (full and by coordinate)
- Events: OnPathCalculated, OnPathFailed
- Public API: RequestPath(), CancelRequest(), InvalidateCache(), InvalidateCacheAt()

### MovementController.cs (399 lines)
- Smooth Vector3.Lerp-based hex-to-hex movement animation
- Path following with step-by-step validation
- Terrain change detection during movement
- Configurable move speed and rotation settings
- 2D sprite support (Z-position preservation, rotation disabled by default)
- Events: OnMovementStart, OnMovementStepStart, OnMovementStep, OnMovementComplete, OnMovementCancelled
- Turn-based pause/resume integration
- Debug Gizmos for path visualization

## Updated Components

### PlayerController.cs
- Integrated MovementController for smooth animation
- Added OnMovementStepStart event handler for immediate path updates
- Real-time committed path visualization (hides traveled segments immediately)
- Removed manual transform.position lerping (~100 lines removed)
- Maintains turn-based AP consumption per cell
- Z-position explicitly set to 0 for proper 2D sprite rendering

## Technical Details

**Performance:**
- Coroutine-based processing prevents frame drops
- Path caching reduces redundant A* calculations
- Configurable throttling limits (nodes/frame, ms/frame)
- Automatic cache cleanup every 60 frames

**Movement System:**
- Frame-rate independent animation
- Preserves Z-position for 2D sprite sorting
- Rotation disabled by default for top-down sprites
- Configurable validation and terrain checking

**Turn Integration:**
- AP consumption on movement completion (not start)
- Auto-pause when AP reaches 0
- Auto-resume when turn starts with refreshed AP
- Compatible with multi-turn pathfinding

**Path Visualization:**
- Committed path updates on OnMovementStepStart
- Shows cells from currentIndex+2 onwards (future destinations only)
- Hides path segment player is currently traveling
- Clean visual feedback throughout movement
