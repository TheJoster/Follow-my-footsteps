feat(npc): implement NPC foundation with hierarchical state machine (Phase 4.1-4.2)

Comprehensive NPC system with data architecture, 8 AI states, and state machine framework.

**NPC Data Architecture** (Phase 4.1):
- NPCDefinition ScriptableObject: Stats, type, vision, initial state, loot table
- NPCRuntimeData serializable class: Health, position, state, inventory, factions
- LootTable ScriptableObject placeholder (Phase 6 implementation)
- NPCType enum: Friendly, Neutral, Hostile

**Hierarchical State Machine** (Phase 4.2):
- IState interface: OnEnter(), OnUpdate(), OnExit() contract
- StateMachine generic class: State registry, transitions, OnStateChanged event
- NPCController MonoBehaviour: Integrates state machine + movement + health

**AI States Implemented** (8 total):
- **Hostile NPCs**: IdleState, PatrolState (Loop/PingPong), ChaseState, FleeState
- **Friendly NPCs**: WanderState (random movement in radius), DialogueState, WorkState
- **Neutral NPCs**: TradeState
- WorkState supports 8 work types: Mining, Farming, Crafting, Guarding, Fetching, Fishing, Cooking, Building

**NPCController Features**:
- Auto-configures states based on NPCType (Friendly/Neutral/Hostile)
- Health management: TakeDamage(), Heal(), automatic flee at 30% health
- State transitions: Manual via ChangeState() or automatic via perception
- Visual appearance: Applies sprite and color tint from definition
- Debug logging: Optional state transition tracking
