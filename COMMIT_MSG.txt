feat(npc): implement NPC spawning system with object pooling (Phase 4.1 complete)

Implement EntityFactory spawning system and NPCSpawner scene component to bring NPCs to life in the game world. NPCs can now be spawned, managed with object pooling, and automatically registered with the turn-based simulation system.

## Phase 4.1 - NPC Spawning & Object Pooling

### EntityFactory.cs (New - 336 lines)
**Spawning System with Object Pooling:**
- `SpawnNPC(NPCDefinition, HexCoord)` - Spawn NPC at position with full initialization
- Object pool management (initial size: 20, max: 100 NPCs)
- `DespawnNPC(GameObject)` - Return NPC to pool, clean up state
- `GetNPC(entityId)` - Retrieve active NPC by unique ID
- `GetAllActiveNPCs()` - Query all active NPCs in scene
- `ClearAll()` - Despawn all NPCs and reset pool

**Unique Entity IDs:**
- Auto-generated IDs: `NPC_{NPCName}_{incrementalID}`
- Ensures each NPC instance is uniquely identifiable
- Supports deterministic turn order and save/load (future)

**Procedural Sprite Generation:**
- `CreateNPCSprite(NPCDefinition)` - Generate colored circle sprites
- Falls back to procedural generation if NPCSprite not assigned
- Uses ColorTint from NPCDefinition (red/blue/yellow for hostile/friendly/neutral)
- Same visual style as PlayerSpawner (128x128, white circle, 32 PPU)

**Cell Occupancy Management:**
- Mark cell as occupied on spawn
- Clear occupancy on despawn
- Validate walkability before spawning (movement cost < 999)

**Component Setup:**
- Auto-adds SpriteRenderer, NPCController, MovementController
- Configures sorting layer ("Entities"), z-position (-1, in front of terrain)
- Scales to 40% of hex size (matching player scale)
- Initializes NPCController with definition and starting position

**Singleton Pattern:**
- Auto-creates instance if not found
- DontDestroyOnLoad for persistence across scenes
- SetHexGrid(HexGrid) for external configuration

### NPCSpawner.cs (New - 279 lines)
**Scene Component for Initial NPC Spawning:**
- `NPCSpawnConfig` serializable class: definition + position + enabled flag
- Auto-spawn on Start (configurable via inspector)
- `SpawnAllNPCs()` - Spawn all configured NPCs from list
- `DespawnAllNPCs()` - Clean up all spawned NPCs

**Auto-Load NPC Definitions:**
- `LoadDefaultNPCSpawns()` - Auto-discovers NPCDefinition assets in editor
- Creates spawn configs with spread-out positions (3-unit horizontal spacing)
- Fallback if no spawn configs manually assigned

**Editor Integration:**
- Gizmos visualization of spawn points (colored spheres + wireframe cubes)
- Color-coded by NPC type: Blue (Friendly), Red (Hostile), Yellow (Neutral)
- Visual feedback for enabled/disabled spawn points

**References:**
- Auto-finds EntityFactory (creates instance if needed)
- Auto-finds HexGrid for coordinate validation
- Tracks spawned NPCs for despawn on component destruction

## Integration

**Automatic SimulationManager Registration:**
- NPCController.Start() already registers with SimulationManager
- Spawned NPCs automatically participate in turn-based simulation
- No additional integration code required

**Usage:**
1. Add NPCSpawner component to scene GameObject
2. NPCs auto-spawn on Start (or call SpawnAllNPCs() manually)
3. NPCs appear in game world with proper visuals
4. NPCs begin executing AI behaviors (Idle, Patrol, Wander, Work)
5. Turn system processes NPC actions each turn

## Technical Decisions

**Object Pooling Strategy:**
- Initial pool: 20 NPCs (sufficient for test scenes)
- Max pool: 100 NPCs (prevents unbounded memory growth)
- Excess NPCs destroyed if pool full on despawn
- Lazy pool expansion (create new if pool exhausted)

**Sprite Generation:**
- Procedural generation ensures NPCs always have visuals
- Consistent with PlayerSpawner implementation
- ScriptableObject sprites can override if assigned
- Color-coded by NPC type for visual clarity

**Cell Validation:**
- Spawn blocked on impassable terrain (movement cost >= 999)
- Spawn blocked on cells outside grid bounds
- Cell marked as occupied to prevent overlapping entities
